There is a real tension in Peers about what am I trying to do. For well over 10 years (probably closer to 15) I have been trying to build an application that's more like a host application for other applications. These other applications are like toy applications. Basically the overhead of a "real" application makes it incredibly unlikely that the toy apps can be written and published, because the application itself is effectively only a few hundred lines. Getting that packaged up into a web application or a mobile or desktop application just doesn't feel worth it.  

You might ask, "So what?"  The issue is that one of the best ways for useful applications to come into existence is for them to start as very small, toy applications. So if the overhead of packaging and deploying a toy application is prohibitively expensive, it stands to reason that this is inhibiting the creation of useful software.

Spreadsheets are the most prolific and used computing technology in the world. There are many really useful data sets in Excel as well as code written (mostly analyses) in excel against those data sets. These never get published because there is no way to publish any kind of "application" written in excel.  The key point here being the most common place where "toy "applications are written provides no mechanism to grow into a larger application. It just has to be completely rewritten. And not in some trivial sense. This rewrite is going to include that large overhead of packaging and deploying, compared to what the developer is used to which is just emailing somebody an Excel file.

So, creating a shell application where very simple little toy applications can be easily deployed and networked, and then slowly built upon, iterated against, and eventually scaled up to a full-fledged, large scale, production applications has been, affectively, a lifelong pursuit of mine.

The issue is I now find myself with this idea, this dream, almost fully realized, but having come about by trying to do something completely different. The question to myself is, "What path do I pursue?" Do I pursue the application I was trying to write, which is a tool for creating configuring, running and interacting with AI assistants? Or now that I have effectively figured out and for the most part already created the shell application that I've been trying to create for the past 10 to 15 years, do I pivot *back* to that? Or do I just continue forward with this Frankenstein application that is trying to be many different things at once?

It's worth mentioning that this Frankenstein application is also trying to be a third thing: A personal knowledge system (PKS) that I believe I have a divine mandate to try to create. If I trust that mandate then obviously that's what I should be doing. But should I also be including these other two aspects (Host application or AI Assistant app) or are they distracting from the PKS?

I believe the PKS relies on the AI assistant portion. And the application host portion seems to be needed to attract users and developers, and to prevent calcification.  In some ways it's just trying to reach parity with Obsidian's plug-ins which is a primary competitor.  

So in conclusion, I'm going to keep going with the Frankenstein version that includes the AI assistant portion, the host application portion, and the PKS. Thanks bye.
