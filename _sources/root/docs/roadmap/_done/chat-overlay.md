
I want to add a global chat overlay to the UI.  

The primary use-case is to let users quickly iterate on packages.  UIs are the key to giving users visibility into what is happening (whether it be UI, tools, etc.) so allowing users to chat with their assistants as they are looking at the UI will enable maximum productive of UI+chat.

A secondary use-case is to be able to view different chat threads regardless of what view/app they are on.

The chat should be a "thread view" (check peers-ui) but overlaid over the current screen instead of users having to jump back and forth to different tabs.  They should be able to select existing threads or start a new one.  When they send a message it should include the current, full UI path to clearly describe what tab/screen they are on so if an assistant (Agent) processes the message it has that context

The minimized view should just be a little floating button in the bottom right of the screen and include a little red indicator if there are unread messages in any of their open threads.
