# 🧠 Frames System — Use Cases Overview

This document summarizes the primary use cases supported by the Frames system.

---

## 1. Structured Reasoning (Core Use Case)

### Description
Build clear, traceable arguments from assumptions to conclusions.

### Capabilities
- Define:
  - observations
  - assumptions
  - values
  - inferences
  - conclusions
- Track dependencies between ideas
- Reuse conclusions across frames

### Outcome
- Clear, inspectable reasoning chains
- Conclusions tied to their supporting premises

---

## 2. Knowledge Base / Thought Graph

### Description
Accumulate reusable propositions and relationships over time.

### Capabilities
- Store blocks globally
- Reuse across multiple frames
- Build a shared reasoning graph

### Outcome
- Growing network of structured knowledge
- Reduced duplication of ideas

---

## 3. Implication Exploration (“What Follows?”)

### Description
Explore consequences of adopting a belief or assumption.

### Capabilities
- Select an assumption
- Traverse downstream implications
- Surface:
  - direct consequences
  - second-order effects
  - conflicts

### Outcome
- Users understand the full impact of beliefs
- Reveals hidden consequences of worldviews

---

## 4. Thesis / Argument Construction (Backward Reasoning)

### Description
Start with a desired conclusion and build supporting arguments.

### Capabilities
- Define a thesis
- Identify required premises
- Surface:
  - missing assumptions
  - weak links
  - required evidence

### Outcome
- Stronger, more complete arguments
- Visibility into what must be true for a claim to hold

---

## 5. Goal / Value Analysis (Is–Ought Separation)

### Description
Separate facts from value judgments and make tradeoffs explicit.

### Capabilities
- Treat goals as first-class blocks
- Detect:
  - missing value bridges
  - implicit value assumptions
- Compare competing values

### Outcome
- Clear distinction between:
  - what is true
  - what matters
- Better understanding of disagreement sources

---

## 6. Goal Reframing

### Description
Challenge or expand the objective being optimized.

### Capabilities
- Replace or expand goals
  - e.g. “reduce gun violence” → “reduce total violence”
- Detect:
  - subset relationships
  - proxy goals
- Re-evaluate conclusions under new goals

### Outcome
- Avoid optimizing the wrong metric
- Reveal hidden tradeoffs

---

## 7. Frame Comparison

### Description
Compare two or more frames to identify differences.

### Capabilities
- Identify:
  - shared propositions
  - disputed propositions
  - differing assumptions
  - differing values
- Compare reasoning paths

### Outcome
- Clear understanding of where arguments diverge
- Reduced confusion in debates

---

## 8. Mediation / Conflict Resolution

### Description
Help multiple parties reach a workable agreement.

### Capabilities
- Represent each party as a frame
- Identify:
  - shared goals
  - constraints / non-negotiables
- Generate candidate resolutions
- Evaluate outcomes across frames

### Outcome
- Structured negotiation
- Identification of viable compromises

---

## 9. Shared Ground Discovery

### Description
Find common agreement across conflicting perspectives.

### Capabilities
- Detect propositions accepted across frames
- Highlight:
  - shared observations
  - shared values (if any)

### Outcome
- Foundation for productive discussion
- Reduced adversarial friction

---

## 10. Assumption Surfacing

### Description
Make implicit assumptions explicit.

### Capabilities
- Identify unstated premises
- Distinguish:
  - observations vs assumptions
- Highlight differences in interpretation

### Outcome
- Clearer reasoning
- Better understanding of disagreement roots

---

## 11. Argument Quality Analysis

### Description
Detect weak or invalid reasoning patterns.

### Capabilities
- Identify:
  - strawman arguments
  - ad hominem attacks
  - unsupported inferences
  - circular reasoning
  - emotional generalizations

### Outcome
- Improved argument quality
- More rational discourse

---

## 12. Reframing & Perspective Shifting

### Description
Allow users to reinterpret arguments under different frames.

### Capabilities
- “Try on” a frame
- Compare alternative interpretations
- Rephrase positions accurately

### Outcome
- Increased empathy and understanding
- Reduced misrepresentation

---

## 13. Multi-Goal Evaluation

### Description
Evaluate conclusions against multiple competing goals.

### Capabilities
- Test a conclusion against:
  - different goals
  - different value systems
- Surface tradeoffs

### Outcome
- Better decision-making
- Explicit tradeoff awareness

---

## 14. Reasoning Traceability

### Description
Track how conclusions are derived.

### Capabilities
- Trace:
  - upstream dependencies
  - supporting assumptions
- Recompute when inputs change

### Outcome
- Transparent reasoning
- Easy debugging of arguments

---

## 15. Evolving Reasoning Over Time

### Description
Allow reasoning to adapt as inputs change.

### Capabilities
- Update assumptions
- Automatically identify affected conclusions
- Re-evaluate frames

### Outcome
- Living, dynamic reasoning system
- Continuous refinement of beliefs

---

## Core Insight

> The system enables users to:
- build arguments
- explore implications
- compare perspectives
- resolve conflicts
- and refine goals

All using a shared structure of:
- neutral propositions (blocks)
- contextual interpretations (frames)

---

## One-Line Summary

A system for structuring, exploring, comparing, and refining reasoning across individual, conflicting, and collaborative contexts.
